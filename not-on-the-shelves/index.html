How Learning Works

  • Started reading the education literature in 2011
  • We know a lot about how people learn and how best to teach them
  • But most faculty don't know this knowledge exists

For Example

  • We have known for 20 years that peer instruction outperforms classic lecturing
  • But no major MOOC platform supports peer instruction
  • No technical reason for the absence

Computer Science Strong Opinion

  • Helped edit "greatest hits" collection on empirical studies in software engineering
  • We know a lot about software and how it's built
  • But students aren't ever shown empirical methods

For Example

  • C-style syntax is as hard to learn as random syntax
  • Python and Ruby are significantly easier…
  • …and A/B-tested languages are even easier
  • But no major language user-tests new features

The Irony of Simplex Communication

  • Hundreds of free-range "learn to code" groups
  • "We will teach them Javascript and CSS…"
  • …and figure out community organization on our own
  • No recognition that knowledge should flow both ways

Is There a Pattern?

  • Smart people stumble for lack of basic skills
  • Nothing on the implementation of tools
  • No description of actual architectures
  • Version control can't handle commonly-used files
  • Most faculty have never met evidence-based pedagogy
  • No major MOOC platform supports peer instruction
  • Students aren't ever shown empirical methods
  • No major language user-tests new features
  • No legal obligation to audit algorithms for bias
  • No recognition that we can learn a lot from them

An Uncomfortable Truth

  1. some smart young people get CS degrees
  2. most never pay attention to CS again

And yes, it's not just a problem in computing

Some Uncomfortable Truths

  • Academics don't respect practitioners' timescales
  • And don't seem to care if practitioners read their work
  • Practitioners don't understand that research isn't about writing software
  • Computer scientists are (also) penalized for writing good code

The Youngest Science

  • 1747: James Lind does the first controlled medical experiment in history
  • 1950: Hill & Doll's study of smokers and non-smokers
  • Finding #1: smoking causes cancer
  • Finding #2: most people would rather fail than change

Like Water on Stone

  • Doctors do science
  • So they know what it has to do with them
  • So they teach it to the next generation
  • Stop asking CS students to pretend to do a project
  • Have them do science instead

Proposal #1

You have been given 80K lines of C and a static analysis tool that can identify lines containing buffer overruns and memory leaks. Determine if complex functions are more likely to contain errors than simple ones.

Requires:

  • Tool use
  • Model building
  • Statistics

And it's intellectually respectable
