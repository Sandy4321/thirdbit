Not on the Shelves

 

Greg Wilson

Software Carpentry Foundation

 

 

September 2016

 

 

What doesn't exist can tell us a lot about what does.

 

 

Warning: I suck at graphical design.

Once Upon a Time…

  • Support programmer in university computing center
  • Saw smart people stumble for lack of basic skills
  • Complained (a lot)
  • Eventually told "put up or shut up"
  • First Software Carpentry class ran at LANL in 1998

Time Passes…

  • Book review editor for Doctor Dobb's Journal
  • Most rehashed the same handful of topics
  • Almost nothing on the implementation of tools
  • Tried some sympathetic magic
  • Didn't lead to many books…
  • …at least, not right away

More Time Passes…

  • Asked to teach a course on software architecture
  • Looked at a couple of dozen books…
  • …none of which described actual architectures
  • So I asked people to describe the most beautiful piece of code they'd ever seen

Rebooted Software Carpentry in 2010

  • Me: "Don't use Word to write your papers."
  • Them: "Why not? It's a lot simpler than LaTeX."
  • Me: "Because version control can't handle it."
  • Them: "Why not? It's just zipped XML."
  • Me: …
  • Them: "Also, what's up with Excel??"

How Learning Works

  • Started reading the education literature in 2011
  • We know a lot about how people learn and how best to teach them
  • But most faculty don't know this knowledge exists

For Example

  • We have known for 20 years that peer instruction outperforms classic lecturing
  • But no major MOOC platform supports peer instruction
  • Absence isn't technical

Computer Science Strong Opinion

  • Helped edit similar collection in 2010 on empirical studies in software engineering
  • We know a lot about software and how it's built
  • But students aren't ever shown empirical methods

For Example

  • C-style syntax is as hard to learn as random syntax
  • Python and Ruby are significantly easier…
  • …and A/B-tested languages are even easier
  • But no major language user-tests new features

The Garbage Fire That Is "Twitter"

  • Harassment is endemic…
  • …unless you're the Olympics or the police
  • Meanwhile, no legal obligation to audit algorithms for racial or gender bias

The Irony of Simplex Communication

  • Hundreds of free-range "learn to code" groups have sprung up
  • "We will teach them Javascript and CSS"
  • No recognition that knowledge should flow both ways
  • Community organization, non-profit management, how to run a meeting…

Is There a Pattern?

  • Smart people stumble for lack of basic skills
  • Nothing on the implementation of tools
  • No description of actual architectures
  • Version control can't handle commonly-used files
  • Most faculty have never met evidence-based pedagogy
  • No major MOOC platform supports peer instruction
  • Students aren't ever shown empirical methods
  • No major language user-tests new features
  • No legal obligation to audit algorithms for bias
  • No recognition that we can learn a lot from them
