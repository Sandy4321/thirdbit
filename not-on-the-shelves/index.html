How Learning Works

  • Started reading the education literature in 2011
  • We know a lot about how people learn and how best to teach them
  • But most faculty don't know this knowledge exists

For Example

  • We have known for 20 years that peer instruction outperforms classic lecturing
  • But no major MOOC platform supports peer instruction
  • No technical reason for the absence

Computer Science

Computer Science Strong Opinion

  • Helped edit "greatest hits" collection on empirical studies in software engineering
  • We know a lot about software and how it's built
  • But students aren't ever shown empirical methods

For Example

  • C-style syntax is as hard to learn as random syntax
  • Python and Ruby are significantly easier…
  • …and A/B-tested languages are even easier
  • But no major language user-tests new features

Like Water on Stone

Proposal #1

Given version control repositories for six software projects, determine whether long functions and methods are more likely to be buggy than short ones.

  • Requires tool use, model building, and statistics
  • Encourages students to do, so they understand and value, so they engage
  • Fits into existing curriculum
  • And it's culturally defensible
